# OpenReader 上游驱动重构计划

## 摘要

本计划以 `hectorqin/reader` 上游项目为基准，重新梳理 OpenReader 的前端与配套后端能力。目标不是继续局部修补单个页面，而是按上游真实产品结构重建：书架工作台、阅读器、书源管理、书籍详情、目录、书签、搜索、设置、本地书库、WebDAV、移动端交互与各模块之间的跳转逻辑。

根目录 `plan.md` 作为执行摘要；更细的上游审阅和组件职责记录放在 `docs/upstream-driven-refactor-plan.md`。

## 核心改造

### 1. 应用结构

- 保留 Vue3 / Vite / Pinia / Element Plus 与 Go REST 技术栈。
- 建立应用级弹窗与抽屉层，承载书籍信息、书籍管理、分组管理、书签、书内搜索、替换规则、用户管理、RSS 等跨页面能力。
- 页面通过统一状态和事件流串联：书架 -> 书籍详情 -> 阅读器 -> 目录 / 书签 / 设置 / 书源切换。

### 2. 书架与首页

- 首页重构为上游式书架工作台，而不是普通列表页。
- 书架支持最近阅读、分组筛选、搜索、更新状态、阅读进度、书籍操作入口。
- 书籍卡片展示封面、书名、作者、最新章节、阅读进度、更新时间、书源/分类状态。
- 批量管理对齐上游：分组、删除、缓存、导出、刷新、换源。
- 移动端首页使用单列结构和底部导航/抽屉式操作，避免桌面侧栏压缩到小屏。

### 3. 搜索与加书流程

- 搜索页对齐上游的“搜索 -> 结果聚合 -> 书籍信息 -> 加入书架/立即阅读”流程。
- 搜索结果按书源聚合展示，支持作者、最新章节、书源名、简介、封面。
- 加书前必须进入书籍信息预览，不能直接跳断流程。
- 缺少后端能力时，相关按钮隐藏或禁用，不做假交互。

### 4. 阅读器

- 阅读器布局参照上游：左侧竖向主工具栏，右侧圆形快捷工具栏，中间固定宽度阅读纸张区域。
- 桌面默认阅读宽度参考上游 `readWidth: 800`，并允许设置调整。
- 左侧工具：返回首页、书架、书源、目录、设置、顶部、底部。
- 右侧工具：书签、书内搜索、书籍信息、笔记/标注、下载/缓存、自动阅读、护眼/夜间、听书。
- 阅读器补齐返回键，返回行为优先回到书架/详情页，直接访问时回首页。
- 所有按钮必须有明确功能、状态和可点击反馈。
- 移动端使用沉浸式正文、点击区域和底部抽屉，不照搬桌面双侧栏。

### 5. 阅读器弹层与组件职责

- 书架弹层：在阅读中快速换书。
- 书源弹层：显示当前书可用书源，支持切源、刷新、加载更多。
- 目录弹层：展示章节列表、当前章节、缓存状态、跳转。
- 阅读设置：主题、字体、字号、字重、行高、段落间距、阅读宽度、翻页/滚动方式、替换规则。
- 书签：添加、编辑、删除、跳转。
- 书内搜索：搜索当前书内容并跳转到匹配章节。
- 书籍信息：封面、作者、简介、目录刷新、分组、换源、加入/移出书架。
- 听书/自动阅读/缓存：如果后端或浏览器能力不足，保留入口状态并明确禁用原因。

### 6. 书源管理

- 书源页对齐上游能力：列表、搜索、分组、启用/禁用、编辑、导入、导出、测试、调试、批量检查。
- 源测试展示搜索、目录、正文链路的具体结果，而不只返回成功/失败。
- 支持远程书源导入、JSON 导入、单源编辑。
- 后端未补齐前，不展示不可用的装饰按钮。

### 7. 书籍详情

- 书籍详情页承担上游 `BookInfo` 的完整职责。
- 展示封面、标题、作者、分类、简介、最新章节、当前书源、阅读进度、目录入口。
- 支持加入书架、继续阅读、刷新目录、换源、分组、删除、下载/缓存。
- 从搜索结果、书架、阅读器信息按钮进入同一套详情逻辑。

### 8. 本地书库、WebDAV 与备份

- 本地书库对齐上游文件管理：目录浏览、上传、删除、导入、恢复。
- WebDAV 对齐上游：连接配置、目录浏览、上传、下载、删除、导入备份。
- 设置页整合账户、备份、WebDAV、阅读默认设置、管理能力。
- 文件类操作必须有明确成功/失败反馈，不能只展示静态卡片。

### 9. 移动端适配

- 移动端不是缩小桌面版，而是单列信息架构。
- 首页使用底部导航或顶部标签，批量操作进入全屏抽屉。
- 阅读器使用沉浸式正文，点击唤起工具栏，目录/设置/书签/书源全部使用底部抽屉。
- 所有按钮保持可触达尺寸，避免侧栏挤占正文。
- 关键流程必须覆盖手机宽度：书架、搜索、详情、阅读、目录、设置、书源。

## 后端接口补齐

### P0 必须补齐

- 书籍更新：编辑书籍、删除书籍、刷新目录、更新单本书。
- 分类管理：编辑、删除、排序、批量设置分组。
- 阅读器能力：书内搜索、可用书源列表、切换书源、章节缓存状态。
- 书签能力：添加、编辑、删除、列表、跳转定位。
- 阅读进度：书架列表返回最近阅读章节、百分比、更新时间。

### P1 继续补齐

- 批量书籍管理：批量删除、批量分组、批量缓存、批量导出。
- 书源增强：导入预览、批量测试、无效源检查、恢复默认、清空。
- 本地书库：目录浏览、上传、删除、导入指定文件。
- WebDAV：浏览、上传、下载、删除、恢复备份。

### P2 高级能力

- 替换规则：独立 CRUD、启用/禁用、规则测试、阅读器实时应用。
- RSS：源管理、文章列表、正文阅读。
- 探索/书海：按书源发现书籍。
- 文件上传：封面、字体、背景图。
- TTS/听书：朗读状态、章节队列、播放进度。

## 实施阶段

### 阶段 0：冻结零散修补

- 停止只针对截图逐点修 UI。
- 以本计划和上游组件职责作为后续重构依据。
- 建立“上游能力 -> 当前实现 -> 缺口 -> 后端接口”的对照表。

### 阶段 1：全局结构与共享组件

- 建立应用级弹窗/抽屉管理。
- 抽出书籍卡片、书籍信息、目录、书签、阅读设置、书源选择等共享组件。
- 统一书籍、章节、书源、分类、书签、阅读配置的前端状态模型。

### 阶段 2：书架、搜索、详情

- 重构首页书架工作台。
- 重构搜索与加书流程。
- 重构书籍详情，并让书架、搜索、阅读器共用同一套详情逻辑。
- 补齐移动端书架与详情布局。

### 阶段 3：阅读器

- 按上游布局重建阅读器视觉结构。
- 补齐左侧工具栏、右侧快捷栏、返回键、底部进度、上一章/下一章。
- 接入目录、书签、书内搜索、书源切换、阅读设置。
- 完成移动端沉浸式阅读交互。

### 阶段 4：管理页

- 重构书源管理页。
- 重构本地书库、WebDAV、备份、设置页。
- 所有按钮都必须接入真实 API 或明确禁用。
- 批量能力等依赖后端的功能按接口补齐顺序推进。

### 阶段 5：高级能力

- 替换规则。
- RSS。
- 探索/书海。
- TTS/听书。
- 自动阅读与缓存管理增强。

## 验收标准

- 桌面端流程可用：书架 -> 搜索 -> 书籍详情 -> 加入书架 -> 阅读 -> 目录 -> 书签 -> 设置 -> 换源。
- 移动端流程可用：书架、搜索、详情、阅读、目录、设置、书签、书源切换均无横向溢出。
- 阅读器视觉接近上游布局：正文宽度合理，左右工具栏语义明确，无无意义按钮。
- 每个可见按钮都有真实功能、禁用状态或明确提示。
- `npm run build` 通过。
- 浏览器人工检查覆盖：`/`、搜索页、书源页、本地书库、设置页、书籍详情、阅读器。
- 后端未实现能力不得以前端假状态冒充完成。

## 默认假设

- 保留当前多路由结构，但产品交互对齐上游的首页工作台和阅读器主流程。
- 不强行一比一复制上游 UI 代码；复制的是信息架构、组件职责、交互逻辑和功能完整度。
- 上游已有但当前后端缺失的能力，先进入接口补齐计划，再进入前端可用实现。
- 移动端作为一等目标处理，不作为桌面布局的附带压缩版本。
